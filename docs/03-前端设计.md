## 2. 核心组件设计

### 2.1 MapViewer 组件

**功能**：Cesium 地图容器，负责 3D 场景初始化、相机信息同步，以及北斗格网单元的点击拾取。

**关键点**：
- Ion Token 可选：未配置时回退到椭球地形 + ESRI 影像
- 点击拾取：中小规模格网用 `scene.pick`；大规模格网调用 `BeiDouGridPrimitive.pick()` 做 GPU 拾取

### 2.2 RegionSelector 组件

**功能**：区域选择组件

**功能点**:
- 城市/地区下拉选择（内置示例）
- 导入 SHP（推荐 zip：`.shp/.dbf/.shx/.prj`），解析 bbox 并提示坐标系风险
- 显示区域边界和面积

**数据格式**:
```javascript
{
  name: String,
  bounds: { minX, minY, maxX, maxY },
  area: Number | null
}
```

### 2.3 StationManager 组件

**功能**：站点管理组件

**功能点**:
- 地图上点击添加站点
- 站点列表管理（增删改查）
- 站点属性编辑（名称、高度、频段）
- 批量导入站点（CSV/GeoJSON）
- 站点可视化标记

**数据模型**:
```javascript
{
  id: String,
  name: String,
  position: { lon, lat, height },
  frequency: Number,      // GHz
  height: Number,          // 绝对海拔（米）
  type: String            // 站点类型
}
```

### 2.4 GridConfig 组件

**功能**：格网配置组件

**配置项**:
- 格网尺寸：宽度(DX)、高度(DY)、层高间隔(DZ)
- 高度范围：最低高度(Z_MIN)、最高高度(Z_MAX)
- 网格类型：规则网格 / 北斗网格
- 自动使用已选区域边界开关
- 北斗格网样式：面颜色/透明度、边线颜色/透明度

**参数验证**:
- DX、DY、DZ > 0.1米
- Z_MIN < Z_MAX
- 范围需大于格网尺寸

### 2.5 AnalysisPanel 组件

**功能**：LOS分析配置面板

**配置项**:
- **基础参数**：
  - 发射端高度字段选择
  - 接收端高度字段选择
  - 频段设置（GHz）
  
- **高级参数**：
  - 是否考虑地球曲率
  - 路径损耗阈值（dB）
  - 批处理大小
  - 最大分析距离

**分析模式**:
- **快速模式**：使用默认参数
- **高级模式**：自定义所有参数

### 2.6 ResultVisualizer 组件

**功能**：分析结果可视化组件

**可视化方式**:
- **点云渲染**：根据可视性着色（绿色=可视，红色=不可视）
- **热力图**：基于可视比例生成热力图
- **统计图表**：可视比例、距离分布等
- **图层控制**：显示/隐藏、透明度调整

**性能优化**:
- 使用Cesium的`PointPrimitiveCollection`批量渲染
- 根据相机距离动态调整点大小
- 支持按层高筛选显示

## 5. 性能要点（北斗格网）

- **中小规模**：`GeometryInstance` 数量在阈值内时，直接构建实例列表，交互拾取简单可靠
- **大规模**：单位几何 + 实例矩阵（Float32Array），使用 GPU 实例化绘制，避免为每个格网单元创建对象导致卡顿/内存暴涨
- **边框透明度表现差异**：大规模 `instanced` 模式不渲染全量边框几何（仅保留面），因此“边框颜色/透明度”只会影响“选中单元高亮”的边框 overlay；如需全量边框，可通过减小网格规模回到 `geometryInstances` 模式
- **拾取**：大规模模式使用离屏渲染 ID 缓冲读回像素，映射为 instanceId（避免遍历/逐个检测）
