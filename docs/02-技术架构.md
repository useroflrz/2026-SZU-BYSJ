## 4. 前端关键链路（当前版本）

### 4.1 北斗格网渲染链路

```
区域选择（城市 / 导入 SHP bbox）
→ 格网参数配置（dx/dy/dz/zMin/zMax + 样式）
→ mapStore.showBeiDouGrid()
→ 规模判断：
   - 中小规模：GeometryInstance + Primitive（可直接 pick）
   - 大规模：BeiDouGridPrimitive（GPU 实例化绘制 + GPU pick）
→ 点击拾取与高亮
```

## 5. 性能优化策略

### 5.1 前端优化
- **分层渲染策略**：中小规模使用 GeometryInstance（交互更直接）；大规模使用自定义 Primitive（GPU 实例化）保证性能
- **实例化绘制**：单位几何 + 实例矩阵数组，减少 DrawCall
- **按批渲染**：单次 instanced draw 受 WebGL 限制，超大实例数分批提交
- **拾取优化**：大规模模式避免逐实例 entity/primitive，使用离屏 ID 缓冲读回实例索引
