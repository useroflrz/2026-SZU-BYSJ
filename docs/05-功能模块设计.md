# 功能模块设计文档（当前版本）

本文件对齐 **当前已实现的前端功能**，重点描述北斗格网渲染链路。后端/分析/报告相关内容不作为当前验收范围。

## 1. 区域选择模块（`RegionSelector.vue`）

- **目标**：产生一个可用于格网生成与渲染的区域范围（bbox）。
- **输入**：
  - 城市/地区：内置示例数据
  - SHP 导入：推荐上传 zip（包含 `.shp/.dbf/.shx/.prj`）
- **输出**：`mapStore.selectedRegion = { name, bounds, area }`
  - `bounds` 结构：`{ minX, minY, maxX, maxY }`（经纬度，WGS84）

## 2. 站点管理模块（`StationManager.vue`）

- **目标**：维护站点数据并渲染到地图（用于展示与后续分析扩展）。
- **输入**：手动/导入（CSV、GeoJSON）
- **输出**：`analysisStore.stations` 与 `mapStore.setStations(stations)`

站点数据结构（简化）：

```javascript
{
  id: String,
  name: String,
  position: { lon, lat, height },
  frequency: Number, // GHz
  type: String
}
```

## 3. 北斗格网生成与渲染模块（核心）

实现入口：`mapStore.showBeiDouGrid(bounds, gridParams)`

### 3.1 参数（来自 `GridConfig.vue`）

- **几何参数（米）**：`dx, dy, dz, zMin, zMax`
- **样式参数**：`fillColor, fillOpacity, outlineColor, outlineOpacity`

### 3.2 两种渲染模式

#### 3.2.1 中小规模：GeometryInstance 模式

- **策略**：为每个格网单元构建 `GeometryInstance`，使用 `Cesium.Primitive + PerInstanceColorAppearance` 渲染。
- **优点**：与 Cesium 原生拾取高度兼容（`scene.pick`）。
- **适用**：总格网数在阈值内（由实现侧判断）。

#### 3.2.2 大规模：Instanced Primitive 模式（`BeiDouGridPrimitive`）

- **策略**：只保留一个单位盒子几何，其余单元通过实例矩阵（instance matrix）在 GPU 端一次性绘制。
- **数据生成**：`createGridInstancesFromBounds(bounds, gridParams)`
  - 输出 `matrixData (Float32Array)`、`batches`、`boundingSphere`
  - 超过单次 instanced draw 上限时按批拆分提交
- **拾取**：`BeiDouGridPrimitive.pick(scene, x, y)`
  - 离屏渲染 instanceId → `readPixels` → 解码得到全局 instanceId
  - 映射到 `beidou-cell-ix-iy-iz`，写入 store 做选中/高亮

## 4. 格网单元选中与高亮（`mapStore.selectBeiDouCell`）

- **中小规模**：直接通过 `getGeometryInstanceAttributes(cellId)` 修改颜色实现高亮
- **大规模**：单独绘制一个高亮盒子 primitive（只渲染选中单元，避免大规模逐实例更新）
