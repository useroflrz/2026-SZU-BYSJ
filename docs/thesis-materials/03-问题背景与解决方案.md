# 问题背景与解决方案

本文归纳系统针对的**工程问题**及对应**实现思路**，便于撰写“需求分析”“总体方案”章节。

---

## 问题 1：桌面三维 GIS 部署与协同成本高

**现象**  
传统桌面软件依赖安装包与特定操作系统，不利于演示、教学与远程协作。

**方案**  
采用 **B/S 架构**：前端为 Vue 3 + Vite 单页应用，三维视图基于 Cesium 在浏览器中运行；分析能力由 **FastAPI** 提供 REST API，前后端可分离部署。

**对应材料**  
[05-架构与数据流（论文叙述版）.md](./05-架构与数据流（论文叙述版）.md)、[docs/07-部署方案.md](../07-部署方案.md)

---

## 问题 2：三维格网规模差异大，单一渲染方式难以兼顾交互与性能

**现象**  
小区域细格网单元数适中时，希望拾取简单、与引擎集成度高；大区域或细步长时，实例数可达百万级，若逐格元构建实体易导致 CPU、内存与 Draw Call 压力。

**方案**  
**规模分流**：低于阈值时用 `GeometryInstance + Primitive`，拾取用 `scene.pick`；超过阈值时用 **自定义 Primitive + GPU 实例化**，拾取用 **离屏 ID 渲染 + readPixels**；超 WebGL 单次实例上限时 **分批 DrawCommand**。实现口径固定为 `MAX_GEOMETRY_INSTANCES=120000`，instanced 单批上限 `65535`。

**对应材料**  
[02-技术亮点与创新点.md](./02-技术亮点与创新点.md)、[docs/02-技术架构.md](../02-技术架构.md)

---

## 问题 3：大规模实例化后，格元级拾取与高亮困难

**现象**  
实例化绘制在引擎语义上常表现为“单个 draw”，原生 pick 难以返回具体实例索引。

**方案**  
**GPU 拾取**：将实例 ID 写入帧缓冲颜色通道，点击时在小视口或离屏通道渲染并解码；**高亮**用独立几何只画当前选中格元，避免批量更新实例缓冲。交互链路为 `MapViewer` 触发点击 -> `primitive.pick` 返回 `instanceId` -> `mapStore.selectBeiDouCell` 同步面板与高亮。

**对应材料**  
[docs/05-功能模块设计.md](../05-功能模块设计.md)、`frontend/src/Rendering/BeiDouGridPrimitive.js`

---

## 问题 4：可视域（LOS）计算耗时长，阻塞请求与界面

**现象**  
格点 × 站点 × LOS 采样计算量大，同步 API 易导致超时或浏览器长时间无响应。

**方案**  
后端 **作业队列 + 状态轮询**：`POST` 创建 job，前端定时查询 `progress`，完成后取 `result`；计算逻辑集中在 `viewshed_engine`，便于单独测试与替换算法。当前前端实现轮询周期约 `500ms`，结果采用紧凑字段 `stats/layerStats/uncoveredIndices` 回写。

**对应材料**  
`backend/main.py`、`backend/app/core/job_manager.py`、`frontend/src/stores/analysis.js`

---

## 问题 5：不规则边界与规则矩形格网不一致

**现象**  
按经纬度 bbox 生成的规则格网会覆盖矩形外区域；分析时若需贴合行政/作业多边形，需要标记无效柱。

**方案**  
**柱掩膜作业**：将裁剪多边形提交后端，生成 `columnActive`（或等价数据），前端用于渲染隐藏/缩放退化及分析输入。前端数据准备阶段会缓存掩膜与地形采样结果，避免同参数重复计算。

**对应材料**  
`backend/main.py`（column-mask）、`frontend/src/stores/map.js`、`frontend/src/utils/gridColumnClip.js`

---

## 问题 6：矢量区域坐标系与 Web 地图不一致导致“格网跑偏”

**现象**  
Shapefile 未使用 WGS84 时，bbox 在 Cesium 上与影像/地形错位。

**方案**  
文档与 UI 侧明确 **推荐 WGS84**；导入时提示坐标系风险；数据预处理在 GIS 软件中完成投影变换（见 [docs/06-数据流程.md](../06-数据流程.md)）。

---

## 问题 7：Cesium 版本与 API 稳定性

**现象**  
高版本 Cesium API 与学校环境或论文复现不一致。

**方案**  
项目约定 **cesium@1.99.0**（见 `frontend/package.json`），实现避免使用更高版本专有 API。

---

## 小结（可直接作“本章小结”素材）

系统从**渲染可扩展性**（双路径 + 实例化 + GPU 拾取）、**分析可伸缩性**（异步作业 + 掩膜裁剪）和**数据一致性**（WGS84 与矢量范围）三方面，支撑浏览器端三维格网与可视域分析的一体化流程。
