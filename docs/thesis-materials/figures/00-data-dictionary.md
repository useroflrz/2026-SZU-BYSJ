# 论文12图统一数据字典（锁定版）

本文件用于锁定图1~图12统一口径。后续出图、图注、正文必须保持一致。

## A. 全局固定口径

- 系统阈值：`N_threshold = 1.2×10^5`（即 120000）
- 状态轮询周期：`500ms`
- 边界偏移对比：本系统 `0像素`；传统方案 `最大2像素`（平均约 `0.7像素`）
- 端到端总耗时：本系统 `11.3s`；传统方案 `42.8s`；节省 `74%`
- 与 ArcGIS 一致率：`94.2%`
- 差异来源拆分：DEM 分辨率差异 `3.1%`；插值算法差异 `2.7%`

## B. 实验场景基础参数

- 研究区域：深圳市南山区（论文口径：高程范围 `0~430m`）
- 站点数量：`5`
- 典型格网分辨率：`100m`
- 性能极端样本：活跃格元约 `1.8×10^6` 时，本系统约 `34.2 FPS`

### 图3 自适应视线采样策略

### 图4 双模式渲染切换机制

- 输入：活跃格元数 `N`
- 判定条件：`N < 1.2×10^5`
- 小规模分支：`GeometryInstance + scene.pick`
- 大规模分支：`GPU Instancing + 离屏ID编码`
- 复杂度标注：绘制调用 `O(N) -> O(1)`

### 图5 系统总体架构图

- 应用层：Vue3、Pinia、Cesium 1.99.0
- 服务层：FastAPI、Uvicorn、计算服务
- 数据层：矢量/地形/格网参数/结果缓存
- 主数据流：输入 -> 格网生成 -> 地形采样 -> 异步计算 -> 结果渲染

### 图6 异步计算任务状态时序图

- 提交分析后返回 `task_id`
- 前端每 `500ms` 轮询 `/status`
- 任务状态字段：`status`, `progress`
- 完成后返回结果索引并重建渲染

### 图7 边界-地形同源机制图
