# 技术亮点与创新点（论文可用表述）

> **表述原则**：突出工程上的**方案组合与实现完整性**（Web 端三维格网 + 规模自适应 + 异步分析），避免夸大“理论首创”。与 [docs/README.md](../README.md) 中弱化后端的表述不一致时，以代码为准。

---

## 1. Web 化三维空域格网可视化

**可写进论文的表述**  
将区域范围、格网步长（米）与高度层参数统一到浏览器端三维场景中，以 WGS84 经纬度范围为基准生成规则三维格网，并支持填充色、透明度与边线样式配置，实现“参数—几何—渲染”的闭环。

**依据**  
- 设计说明：[docs/01-项目概述.md](../01-项目概述.md)、[docs/05-功能模块设计.md](../05-功能模块设计.md)  
- 实现：`frontend/src/stores/map.js`、`frontend/src/components/GridConfig.vue`

---

## 2. 按规模自适应的双路径渲染与拾取

**可写进论文的表述**  
针对格网单元数量跨越多个数量级的特点，采用阈值分流：**中小规模**使用 Cesium `GeometryInstance` 与 `Primitive`，直接利用 `scene.pick` 获取格元标识；**大规模**切换为单几何体加实例矩阵的 **GPU 实例化（instancing）** 绘制，将 Draw Call 与 CPU 侧对象数量控制在可接受范围，并通过分批提交规避单次实例数上限。

**依据**  
- [docs/02-技术架构.md](../02-技术架构.md)、[docs/06-数据流程.md](../06-数据流程.md)  
- 实现：`frontend/src/stores/map.js`、`frontend/src/Rendering/BeiDouGridPrimitive.js`  
- 关键口径：`MAX_GEOMETRY_INSTANCES=120000`（自动分流阈值）、`MAX_BATCH_INSTANCES=65535`（instanced 单批上限）

---

## 3. 大规模模式下的 GPU 拾取与高亮策略

**可写进论文的表述**  
在实例化模式下，传统 `scene.pick` 往往难以直接映射到具体格元。系统通过离屏渲染将 **实例 ID 编码为颜色**，读回像素后解码得到全局实例索引，再映射为 `(ix, iy, iz)` 与业务 `cellId`。高亮采用**单独 Primitive 仅绘制选中格元**，避免对海量实例属性做逐帧更新。

**依据**  
- [docs/05-功能模块设计.md](../05-功能模块设计.md)  
- 实现：`BeiDouGridPrimitive.pick`（`frontend/src/Rendering/BeiDouGridPrimitive.js`）、`frontend/src/components/MapViewer.vue`、`mapStore.selectBeiDouCell`（`frontend/src/stores/map.js`）  
- 关键链路：`scene.pick` 失败时回退 `primitive.pick` -> `instanceId` 反解 `(ix,iy,iz)` -> 独立高亮 primitive

## 4. 后端异步作业驱动的格网可视域（LOS）分析

**可写进论文的表述**  
可视域计算为 CPU 密集型任务，若同步阻塞 HTTP 请求将影响并发与体验。系统将分析封装为**作业（job）**：创建任务返回 `jobId`，前端轮询状态与进度，完成后拉取结果。计算在规则三维格网上对每个有效格点与候选发射站进行 LOS 判定，并结合地形高程栅格做通视判断。

**依据**  
- API：`backend/main.py`（`/api/v1/analysis/jobs` 等）  
- 核心算法：`backend/app/core/viewshed_engine.py`  
- 前端：`frontend/src/stores/analysis.js`、`frontend/src/Analysis/apiClient.js`  
- 关键口径：前端轮询周期约 `500ms`；结果回写采用 `stats/layerStats/uncoveredIndices`

## 5. LOS 路径上的工程化细节（可作为“算法实现”小节要点）

**可写进论文的表述**  
- 接收点高度结合相对层高与地面高程；地面高程对平面位置采用**双线性插值**。  
- LOS 采样点数随距离**自适应**，并设最小/最大采样数边界。  
- 对每个格柱仅在**最大作用距离内**筛选候选站点，减少无效 LOS 次数。  
- 支持按柱激活掩膜（`columnActive`），跳过无效柱，降低计算量。

**依据**  
- `backend/app/core/viewshed_engine.py`（`_bilinear_ground_height`、`_has_line_of_sight`、`run_grid_viewshed_analysis`）

---

## 6. 区域与格网柱的掩膜（前后端协同）

**可写进论文的表述**  
对不规则行政/作业边界，可在平面格网上生成**柱级掩膜**（哪些 `(ix,iy)` 柱参与竖直方向格网与分析），后端提供柱掩膜异步作业，前端将结果用于渲染裁剪或分析输入，使三维格网与分析范围与矢量边界一致。

**依据**  
- `backend/main.py`（`/api/v1/grid/column-mask/jobs`）  
- `frontend/src/Analysis/apiClient.js`（`runColumnMaskJobAndWait`）、`frontend/src/stores/map.js`  
- 关键口径：前端优先复用后端掩膜结果，失败时回退前端裁剪；数据集缓存键包含掩膜哈希避免重复准备

---

## 7. 数据接入与坐标系风险说明（可放在“实现注意事项”）

**可写进论文的表述**  
区域可通过内置城市示例或上传 Shapefile（推荐 zip 含 `.prj`）解析外包矩形；系统以 **WGS84 / EPSG:4326** 为主进行范围与渲染。若源数据为非经纬度投影，需先转换，否则将出现格网偏移（与 [docs/06-数据流程.md](../06-数据流程.md) 常见问题一致）。
