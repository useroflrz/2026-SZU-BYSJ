## 关键输入输出映射（建议写入“详细设计”）

| 模块 | 关键输入 | 关键输出 | 下游消费者 |
|---|---|---|---|
| `GridConfig.vue` | `bounds`、`dx/dy/dz`、`zMin/zMax`、颜色透明度、`renderModeOverride` | 调用 `mapStore.showBeiDouGrid(...)`，形成 `beiDouGridMeta` 与当前渲染层 | `MapViewer.vue`、`AnalysisPanel.vue` |
| `stores/map.js` `prepareBeiDouGridDataset` | bounds、步长、高度范围、`clipGeoJson`/`columnActive` | `gridX/gridY/gridZ`、`groundHeights`、`columnActive`、缓存键 | `stores/analysis.js` 请求组包、`ResultVisualizer.vue` |
| `MapViewer.vue` | 场景点击事件、相机状态、当前渲染模式 | 选中格元 ID/坐标信息、高亮状态同步 | `BeiDouCellInfoPanel.vue`、`stores/map.js` |
| `stores/analysis.js` | `stations`、`gridMeta`、`params(maxDistance/clearance/losSamples...)` | `jobId`、`status/progress`、`gridViewshedCompact(stats/layerStats/uncoveredIndices)` | `AnalysisPanel.vue`、`ResultVisualizer.vue`、`stores/map.js` |

## 模块间数据契约（最小闭环）

- 区域与格网契约：`RegionSelector -> mapStore.setRegion -> GridConfig -> mapStore.showBeiDouGrid`。
- 分析契约：`analysisStore` 始终读取 `mapStore.beiDouGridMeta` 作为 `gridMeta` 来源，避免参数口径分裂。
- 结果契约：分析完成后由 `gridViewshedCompact.uncoveredIndices` 驱动 `mapStore.renderBeiDouUncoveredGridFromCompactResult`，不直接替换原始格网数据。
- 实验契约：`GridPerformanceLab` 与 `SignalStrengthLab` 复用同一格网基础数据，保证实验项之间可比。

## 后端模块（论文可写成“服务层 / 领域逻辑”）

| 模块 | 路径 | 职责摘要 |
|------|------|----------|
| API 入口 | `backend/main.py` | 路由：分析 job、柱掩膜 job、健康检查 |
| 分析作业 | `backend/app/core/job_manager.py` | 创建任务、执行、状态与结果存储 |
| 可视域引擎 | `backend/app/core/viewshed_engine.py` | 格网 LOS 计算、进度回调 |
| 掩膜作业 | `backend/app/core/mask_job_manager.py` | 柱掩膜异步任务 |
| 模型 | `backend/app/models/*.py` | 请求/响应 Pydantic 模型 |
