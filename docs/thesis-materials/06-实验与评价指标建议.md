# 实验与评价指标建议

本文件**不提供虚构数据**，只给出可操作的实验设计与记录项，供撰写“实验”“测试”“系统评价”章节时采用。

---

## 1. 实验环境（建议如实填写）

记录项示例：

- CPU 型号、核心数；内存容量  
- GPU 型号、驱动版本；浏览器类型与版本  
- 操作系统；前端 `npm run build` 与后端 `uvicorn` 版本  
- Cesium 版本：**1.99.0**（见 `frontend/package.json`）

---

## 2. 格网渲染性能实验

**目的** 验证规模分流策略在不同格元数量下的帧率与可操作性。

**代码基线（建议固定）**

- 实验组件：`frontend/src/components/GridPerformanceLab.vue`
- 默认垂向参数：`dz=100m`、`zMin=0`、`zMax=2000m`（20 层）
- 目标规模序列：5000、20000、50000、100000、500000、1000000、2000000、3000000、5000000、8000000
- 时间窗口：默认预热 2s，采样 10s（均可在界面改）
- 跳过规则：目标规模 `>=2000000` 时仅测 instanced（Geometry 记为 `-`）

**自变量建议**

- 区域大小（或 bbox 面积）  
- `dx, dy, dz` 组合（改变总格元数）  
- 渲染模式（若可强制或观察日志/断点：GeometryInstance vs Instanced）

**因变量建议**

- 平均 FPS 或帧时间（Chrome Performance / FPS meter）  
- 首帧格网出现耗时（从点击“显示”到几何稳定）  
- 内存占用（浏览器任务管理器或 Performance memory）

**记录表头示例**

| 场景编号 | 格元约数 | 模式 | 平均 FPS | 备注 |
|----------|----------|------|----------|------|

**采集流程建议（与组件一致）**

1. 清理旧图层并等待渲染稳定；
2. 生成本轮格网并等待预热窗口结束；
3. 在采样窗口内使用 `requestAnimationFrame` 计帧并计算平均 FPS；
4. 导出 Markdown/CSV 原始表，作为论文附录原始记录。

---

## 3. 拾取正确性实验

**目的** 验证中小规模 `scene.pick` 与大规模 GPU pick 结果一致、无系统性偏移。

**步骤建议**

- 选取若干已知 `(ix,iy,iz)` 或通过 UI 能唯一识别的格元  
- 旋转相机至不同俯仰角、缩放级别，重复点击同一屏幕位置附近  
- 记录返回 `cellId` 是否稳定；大规模模式下抽查边界格元

**记录** 异常帧（深度冲突、透明叠加）时的现象与相机参数。

---

## 4. 可视域分析作业实验

**目的** 评价异步作业端到端耗时与进度反馈是否合理。

**自变量建议**

- `gridX × gridY × gridZ`（总格点数）  
- 站点数量、 `maxDistance`  
- `losSamplesMin` / `losSamplesMax`、`clearance`  
- 是否启用 `columnActive` 掩膜及激活比例

**因变量建议**

- 作业总耗时（创建 job 到 result 可用）  
- 进度曲线是否单调、是否出现长时间停滞  
- 结果规模（返回 JSON 大小、前端解析耗时）

**记录** 后端日志中 `viewshed` 相关起止时间（若启用）。

**请求字段口径（建议固定）**

- `gridMeta`：`gridX/gridY/gridZ`、`dx/dy/dz`、`originLon/originLat`、`groundHeights`、`columnActive`
- `params`：`maxDistance`、`clearance`、`losSamplesMin`、`losSamplesMax`、`progressBatchCells`
- 轮询节奏：前端约每 `500ms` 查询一次作业状态

---

## 5. 柱掩膜实验

**目的** 验证矢量裁剪后，渲染与分析仅作用于多边形内柱。

**步骤建议**

- 同一 bbox 下对比：无掩膜 vs 有掩膜  
- 目视检查格网是否被裁切；分析结果点数是否与有效柱数一致

---

## 6. 正确性对比（可选，难度较高）

若有参考实现（如 QGIS viewshed 插件、自写脚本在小网格上暴力验证），可选取**极小网格**（如 10×10×3）对比可见点集合或统计量。需在论文中说明差异来源（高程来源、LOS 采样、地球曲率是否忽略等）。

---

## 7. 论文中可写的结论类型（需基于你的实测）

- 在某硬件与浏览器下，系统可流畅交互的最大约略格元量级（给出前提条件）  
- 异步作业相对同步接口在“界面无卡死”方面的优势（定性 + 耗时数据）  
- GPU 拾取与原生 pick 在选中率上的一致性（定性描述 + 少量统计）

## 8. 数据真实性与可追溯要求

- 只使用界面导出表与日志统计，不手工改写原始帧率/耗时。
- 表格中同时保留“目标格元总数”和“实际格元总数”，并解释差异来自 `columnActive` 裁剪。
- 对于未完成实测项，显式标注“待补实测”，不要写推测性结论。
