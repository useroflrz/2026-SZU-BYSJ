# 数据流程（当前版本）

本文件仅描述 **当前前端实现** 的主流程：区域选择 → 格网参数 → 渲染 → 拾取/高亮。

## 1. 主流程概览

```
用户选择区域（城市 / 导入 SHP）
→ 生成 bounds（WGS84，经纬度 bbox）
→ 配置格网参数（dx/dy/dz/zMin/zMax + 样式）
→ mapStore.showBeiDouGrid(bounds, gridParams)
→ Cesium 场景渲染（按规模选择渲染模式）
→ 点击拾取格网单元 → 选中/高亮
```

## 2. 数据对象定义

### 2.1 区域（Region）

```json
{
  "name": "深圳市",
  "bounds": { "minX": 113.75, "minY": 22.44, "maxX": 114.62, "maxY": 22.86 },
  "area": 1997.47
}
```

### 2.2 北斗格网参数（GridParams）

```json
{
  "dx": 100,
  "dy": 100,
  "dz": 50,
  "zMin": 0,
  "zMax": 500,
  "fillColor": "#00e0ff",
  "fillOpacity": 0.25,
  "outlineColor": "#00e0ff",
  "outlineOpacity": 0.6
}
```

### 2.3 格网元 ID（CellId）

```
beidou-cell-<ix>-<iy>-<iz>
```

## 3. 渲染模式分流

### 3.1 中小规模（GeometryInstance）

- **生成**：三重循环构造 `GeometryInstance[]`
- **拾取**：`scene.pick()` 返回 `picked.id`，直接得到 `cellId`
- **高亮**：`primitive.getGeometryInstanceAttributes(cellId)` 更新颜色

### 3.2 大规模（Instanced）

- **生成**：`createGridInstancesFromBounds()` 输出实例矩阵数组 + 按批拆分数据
- **渲染**：`BeiDouGridPrimitive.update(frameState)` 推入 `DrawCommand`
- **拾取**：
  1. 点击时先尝试 `scene.pick()`（通常不返回 cellId）
  2. 调用 `BeiDouGridPrimitive.pick(scene, x, y)`
  3. 解码得到 `globalInstanceId` 并映射为 `(ix,iy,iz)` → `cellId`
- **高亮**：创建/更新一个单独的高亮 primitive（仅渲染选中格网元）

## 4. 常见问题（与流程相关）

- **SHP 导入后格网位置不对**：可能不是 WGS84（经纬度）坐标；请先将 shp 投影到 WGS84 再导入
- **大区域 + 小 dx/dy 导致卡顿**：会触发大规模渲染模式；仍建议适当增大 dx/dy 或缩小区域以减少实例数
