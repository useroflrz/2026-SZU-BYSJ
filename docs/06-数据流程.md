# 数据流程文档

## 1. 整体数据流程

```
用户操作 → 前端请求 → 后端处理 → 数据存储 → 结果返回 → 前端渲染
```

## 2. 区域选择流程

### 2.1 流程图

```
开始
  ↓

  ↓

  ↓
后端验证区域
  ↓

  ↓
返回区域信息和DSM URL
  ↓

  ↓

```

## 3. 格网生成流程

### 3.1 流程图

```
开始
  ↓
用户配置格网参数
  ↓
前端发送生成请求
  ↓
后端创建异步任务
  ↓
返回任务ID
  ↓
前端轮询/WebSocket监听进度
  ↓
后端处理：
  - 读取DSM数据
  - 生成格网点坐标
  - 采样DSM高程
  - 计算绝对高程
  ↓
生成完成，返回结果
  ↓
前端下载/渲染格网点
  ↓
完成
```

### 3.2 数据转换

**坐标转换**:
```
WGS84 (lon, lat) 
  → 投影坐标 (x, y) 
  → DSM像素坐标 (row, col)
  → 采样高程值
  → 计算绝对高程 (z_abs = dsm_z + z_layer)
```

**格网点数据结构**:
```json
{
  "x": 500000.0,
  "y": 2500000.0,
  "z": 225.5,
  "dsm_z": 125.5,
  "z_layer": 100.0,
  "z_abs": 225.5,
  "grid_x_id": 0,
  "grid_y_id": 0,
  "layer_id": 0
}
```

### 3.3 性能优化

- **分块处理**：大区域分块生成，避免内存溢出
- **并行计算**：多进程处理不同高度层
- **进度反馈**：实时推送处理进度

## 4. LOS分析流程

### 4.1 流程图

```
开始
  ↓
加载格网点和站点数据
  ↓
配置分析参数
  ↓
前端发送分析请求
  ↓
后端创建分析任务
  ↓
WebSocket连接建立
  ↓
批量处理接收点：
  For each receiver:
    - 计算到所有发射源的距离
    - 按距离排序
    - 遍历发射源（从近到远）
      - 路径损耗预过滤
      - LOS视线检查
      - 找到可视即停止
    - 记录结果
  ↓
实时推送进度
  ↓
分析完成，返回结果
  ↓
前端可视化渲染
  ↓
完成
```

### 4.2 数据流

**输入数据**:
- 格网点（接收端）：`[{x, y, z_abs, ...}, ...]`
- 站点（发射端）：`[{x, y, height, frequency, ...}, ...]`
- DSM数据：`{array, extent, cell_size, ...}`

**处理过程**:
```python
for receiver in receivers:
    # 1. 计算距离
    distances = calculate_distances(receiver, transmitters)
    
    # 2. 排序
    sorted_tx = sort_by_distance(transmitters, distances)
    
    # 3. 检查LOS
    for tx in sorted_tx:
        if check_los(receiver, tx, dsm_data):
            result = {
                'receiver_id': receiver.id,
                'visible': True,
                'distance': distance,
                'closest_tx': tx.id
            }
            break
```

**输出数据**:
```json
{
  "receiver_id": "point_001",
  "visible": true,
  "distance": 5000.0,
  "loss_db": 105.2,
  "closest_tx": "station_001",
  "curvature_m": 0.5
}
```

## 5. 可视化渲染流程

### 5.1 数据准备

**前端接收结果**:
```javascript
const results = await fetch('/api/v1/analysis/result/analysis_123')
const data = results.data
```

**数据转换**:
```javascript
// 将投影坐标转换为经纬度
const points = data.results.map(point => ({
  lon: projectToWGS84(point.x, point.y).lon,
  lat: projectToWGS84(point.x, point.y).lat,
  height: point.z,
  visible: point.visible,
  distance: point.distance
}))
```

### 5.2 渲染流程

```
接收分析结果
  ↓
数据转换（投影坐标 → 经纬度）
  ↓
创建Cesium实体：
  - 点云图层（PointPrimitiveCollection）
  - 站点标记（Entity）
  - 热力图图层（可选）
  ↓
设置样式：
  - 可视点：绿色
  - 不可视点：红色
  - 点大小、透明度
  ↓
添加到场景
  ↓
相机飞行到结果区域
```

### 5.3 性能优化

- **分块渲染**：大规模点云分块加载和渲染
- **LOD机制**：根据相机距离调整渲染精度
- **视锥剔除**：只渲染视野内的点

## 7. 数据存储策略

### 7.1 临时数据

- **Redis缓存**：任务状态、进度信息
- **内存存储**：当前会话的站点、格网数据

### 7.2 持久化数据

- **文件存储**：
  - DSM数据：GeoTIFF格式
  - 格网结果：GeoJSON/GeoPackage
  - 分析结果：GeoJSON/CSV
  - 报告文件：PDF/Excel/HTML

### 7.3 数据清理

- **自动清理**：超过7天的临时数据自动删除
- **手动清理**：用户可手动删除不需要的数据

## 8. 错误处理流程

### 8.1 错误类型

- **参数错误**：返回400，提示具体错误字段
- **数据错误**：返回400，提示数据问题
- **处理错误**：返回500，记录日志
- **超时错误**：返回408，提示重试

### 8.2 错误恢复

- **任务失败**：允许重新运行
- **数据丢失**：从备份恢复或重新生成
- **网络中断**：支持断点续传
