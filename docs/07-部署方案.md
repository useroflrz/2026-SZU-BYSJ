# 部署方案文档

## 1. 部署架构

### 1.1 整体架构

采用前后端分离的部署架构，前端静态资源通过Nginx提供服务，后端API服务独立部署，支持水平扩展。

### 1.2 组件说明

- **前端服务**：Vue应用构建后的静态文件，由Nginx提供HTTP服务
- **后端API服务**：FastAPI应用，处理业务逻辑和GIS计算
- **任务队列**：Celery Worker处理异步任务
- **消息队列**：Redis作为Celery的Broker和结果存储
- **数据存储**：文件系统存储DSM数据和结果文件
- **WebSocket服务**：实时推送任务进度

## 2. 环境要求

### 2.1 服务器要求

**最低配置**：
- CPU: 4核
- 内存: 8GB
- 磁盘: 100GB SSD
- 操作系统: Ubuntu 20.04 LTS / CentOS 7+

**推荐配置**：
- CPU: 8核+
- 内存: 16GB+
- 磁盘: 500GB+ SSD
- 操作系统: Ubuntu 22.04 LTS

### 2.2 软件依赖

**系统软件**：
- Python 3.9+
- Node.js 16+ (仅开发环境需要)
- Nginx 1.18+
- Redis 6.0+
- GDAL 3.5+ (包含Python绑定)

**Python包**：
- FastAPI 0.100+
- Uvicorn (ASGI服务器)
- GDAL Python绑定
- NumPy, GeoPandas
- Celery
- Redis客户端

## 3. 部署步骤

### 3.1 前端部署

**构建步骤**：
1. 安装Node.js和npm
2. 进入前端目录，安装依赖
3. 配置环境变量（API地址等）
4. 执行构建命令生成静态文件
5. 将构建产物复制到Nginx静态目录

**Nginx配置**：
- 配置静态文件服务
- 配置API代理转发到后端服务
- 配置WebSocket代理
- 配置HTTPS（可选）

### 3.2 后端部署

**安装步骤**：
1. 安装Python和系统依赖（GDAL等）
2. 创建虚拟环境
3. 安装Python依赖包
4. 配置环境变量和配置文件
5. 使用systemd或supervisor管理服务

**服务管理**：
- 使用Uvicorn运行FastAPI应用
- 配置进程守护和自动重启
- 配置日志轮转

### 3.3 任务队列部署

**Celery Worker部署**：
1. 安装Celery和Redis
2. 配置Celery Worker
3. 启动多个Worker进程（根据CPU核心数）
4. 配置进程监控和自动重启

**Redis配置**：
- 配置持久化（RDB + AOF）
- 设置内存限制
- 配置密码认证

### 3.4 数据存储配置

**目录结构**：
- DSM数据目录：存储GeoTIFF文件
- 结果文件目录：存储生成的格网和分析结果
- 报告文件目录：存储生成的报告
- 临时文件目录：存储处理过程中的临时文件

**权限设置**：
- 确保应用有读写权限
- 配置定期清理临时文件

## 4. 配置说明

### 4.1 前端配置

**环境变量**：
- API_BASE_URL: 后端API地址
- WS_BASE_URL: WebSocket地址
- MAP_TOKEN: Cesium访问令牌

### 4.2 后端配置

**配置文件项**：
- 数据库连接（如使用）
- Redis连接信息
- 文件存储路径
- 任务队列配置
- 日志级别和路径
- CORS允许的域名

### 4.3 性能调优

**Uvicorn配置**：
- Worker数量：建议为CPU核心数
- 连接数限制
- 超时设置

**Celery配置**：
- Worker并发数
- 任务优先级
- 结果过期时间

### 5.2 数据备份

- DSM数据定期备份

## 6. 安全配置

### 6.1 网络安全

- 配置防火墙规则
- 使用HTTPS加密传输
- 限制API访问IP（如需要）

### 6.2 应用安全

- 配置CORS白名单
- 实现API认证（JWT Token）
- 文件上传大小和类型限制
- 输入参数验证和过滤

### 6.3 数据安全

- 敏感数据加密存储
- 定期更新依赖包（安全补丁）
- 配置日志脱敏

## 7. 容器化部署（可选）

### 7.1 Docker部署

**优势**：
- 环境一致性
- 易于扩展
- 简化部署流程

**容器组成**：
- 前端容器（Nginx）
- 后端API容器
- Celery Worker容器
- Redis容器

### 7.2 Docker Compose

使用Docker Compose编排多个容器，简化部署和管理。

### 7.3 Kubernetes部署（可选）

对于大规模部署，可以使用Kubernetes进行容器编排和自动扩缩容。

### 9.1 前端优化

- 启用Gzip压缩

### 9.2 后端优化

- 使用连接池
- 启用响应缓存
- 优化数据库查询（如使用）
- 使用异步处理耗时任务

### 9.3 系统优化

- 调整系统参数（文件描述符限制等）
- 使用SSD存储
- 配置合适的交换空间
